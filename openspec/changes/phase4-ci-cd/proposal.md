# Phase 4: CI/CD 持续集成

## 背景

代码变更缺乏自动化验证门禁：
- PR 合并前没有自动跑测试
- 无法快速发现回归问题
- 覆盖率趋势无法追踪
- 多 Python 版本兼容性（3.8/3.10/3.12）无人验证

## 目标

接入 GitHub Actions，实现：
- 每次 push 和 PR 自动运行全套测试
- 多 Python 版本矩阵验证
- 覆盖率报告自动生成并上传
- 快速（慢速）测试分离，PR 反馈及时

## 非目标

- 不部署到生产环境（CD 部分）
- 不配置代码 lint/format 检查（可单独处理）
- 不购买付费 CI 服务

## 成功标准

- PR 页面可看到测试结果（✅ / ❌）
- 覆盖率报告作为 artifact 可下载
- 快速测试在 3 分钟内完成（排除 `slow` 标记）
- 支持 Python 3.8 和 3.10
