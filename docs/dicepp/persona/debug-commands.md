# Persona AI 调试命令

> `.pa` 命令详细使用说明

## 命令概述

`.pa` (Persona Admin) 命令仅管理员可用，用于调试和管理 Persona AI 系统。

## 命令列表

### 关系管理

#### `.pa rel <用户ID> [群组ID]`
查看指定用户的好感度详情。

**示例：**
```
.pa rel 123456789
.pa rel 123456789 987654321
```

**输出示例：**
```
=== 用户 123456789 的关系详情 ===

[好感度]
  亲密度: 45.50
  激情: 42.00
  信任: 48.25
  安全感: 44.75
  综合: 45.20
  互动次数: 15
  最后互动: 2026-04-10 15:30
  等级: 3 (友好)
```

#### `.pa setrel <用户ID> <综合分数> [群组ID]`
手动设置用户的好感度。

**示例：**
```
.pa setrel 123456789 60
.pa setrel 123456789 80 987654321
```

**说明：**
- 分数范围：0-100
- 设置后所有四维分数相同

### 系统管理

#### `.pa reload`
热重新加载角色卡。

**说明：**
- 重新从磁盘加载角色卡 YAML 文件
- 无需重启 Bot
- 用于角色卡修改后的快速更新

### 日记和事件查看

#### `.pa today`
查看今天的日记和事件。

**输出示例：**
```
=== 今天 (2026-04-10) ===

[日记]
今天的工作还算顺利，虽然有个作者拖稿了...（日记内容）

[事件] (3 个)
  1. [scheduled] 早上泡了杯咖啡，开始处理昨天的稿件
     反应: 咖啡很香，希望能保持清醒高效
  2. [scheduled] 中午和同事去楼下新开的餐厅吃饭
     反应: 味道一般，但聊得很开心
  3. [scheduled] 下午收到了读者来信
     反应: 很感动，觉得工作有意义
```

#### `.pa yesterday`
查看昨天的日记和事件。

---

### 紧急控制

#### `.pa pause`
暂停所有主动消息发送。

**说明：**
- 立即停止主动消息调度器
- 不影响被动响应（@bot 和 .ai 命令）
- 用于紧急情况下停止打扰

#### `.pa resume`
恢复主动消息发送。

## 使用场景

### 场景1：调试好感度系统
```
# 查看当前用户的好感度
.pa debug

# 查看特定用户详情
.pa rel 123456789

# 手动调整好感度测试不同区间
.pa setrel 123456789 25  # 测试"普通"区间
.pa setrel 123456789 55  # 测试"友好"区间
.pa setrel 123456789 75  # 测试"亲近"区间
```

### 场景2：排查主动消息问题
```
# 查看调度器状态
.pa debug

# 查看今天生成的事件
.pa today

# 暂停主动消息（如果太频繁）
.pa pause

# 恢复主动消息
.pa resume
```

### 场景3：更新角色卡
```
# 修改角色卡文件后
.pa reload

# 验证事件配置
.pa events
```

---

## 注意事项

1. **权限控制**：所有 `.pa` 命令仅对管理员（config.admin 或 config.master）有效
2. **生产环境**：谨慎使用 `.pa setrel`，这会直接修改用户数据
3. **调试开关**：`.pa pause/resume` 可以临时控制主动消息，但重启后会恢复配置文件的设置
